# AGENTS.md

This file applies to ToS doctrine under `ToS/doctrine/`.

## Role

`ToS/doctrine/` owns ToS knowledge law: knowledge model, node contract, route
doctrine, template discipline, growth posture, boundary notes, export posture,
and source-first review law.

It is not the decision lane. Durable rationale lives in `docs/decisions/`.

## Read First

1. Root `AGENTS.md`
2. `ToS/AGENTS.md`
3. `ToS/README.md`
4. `ToS/doctrine/KNOWLEDGE_MODEL.md`
5. `ToS/doctrine/NODE_CONTRACT.md`
6. The exact doctrine surface being edited

## Boundaries

- Keep doctrine stronger than review notes and generated readers.
- Keep doctrine lower than primary witness text when it describes a source
  route.
- Keep doctrine from becoming backlog, runtime control, or downstream KAG
  substrate policy.
- Use `docs/decisions/` for durable rationale when a route choice needs to be
  findable across future refactors.

## Validation

Use `ToS/doctrine/REVIEW_CHECKLIST.md` for boundary-sensitive doctrine changes.
Run the route, canon, contract, or export validator named by the surface you
touch.
