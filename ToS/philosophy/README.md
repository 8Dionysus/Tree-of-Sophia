# Philosophy

`ToS/philosophy/` is the growing domain tree of philosophy.

Philosophical branch growth lives here. Temporary extraction routes through
`ToS/candidate-intake/`; research scaffolds route through
`ToS/research-packets/`.

This branch grows the authored domain body from trunk law into eras, regions,
traditions, corpora, archives, inscriptions, works, figures, concepts,
institutions, media, scripts, transmissions, controversies, source anchors, and
local graph fragments.

## Shape

```text
philosophy/
  trunk/
  eras/
  threads/
  graph-workbench/
```

`trunk/` holds branch-wide law: evidence status, chronology posture, and naming
rules. Its current vocabulary homes are:

```text
trunk/
  branch-law/
  chronology/
  evidence-status/
  graph-layers/
  node-kinds/
  relation-kinds/
  source-posture/
```

`eras/` is the primary growth body. A typical mature branch should grow like:

```text
eras/<era>/
  regions/<region>/
    traditions/<tradition>/
      corpora/
      archives/
      inscriptions/
      works/
      figures/
      concepts/
      institutions/
      media/
      scripts/
      transmissions/
      controversies/
      sources/
      graph-workbench/
```

Corpora, archives, inscriptions, institutions, media, scripts, transmissions,
and controversies are first-class philosophical infrastructure. They are not
secondary notes beneath authors and works.

`threads/` holds cross-branch navigation for figures, works, concepts, source
corpora, and transmissions. Threads are navigational organs, not the home of
the object.

`graph-workbench/` holds proposed nodes, proposed relations, branch fragments,
view route cards, and promotion ledgers before any object is raised into
`ToS/canon/`.

## Promotion Route

```text
research packet
  -> branch review
    -> local branch skeleton
      -> proposed nodes
        -> proposed relations
          -> relation pack
            -> canon promotion
              -> derived graph/export
```

The route keeps raw scaffold, branch interpretation, candidate node, candidate
relation, reviewed relation pack, canon, and derived export separate.

## Naming

Use names that describe the philosophical topology:

- `bronze-age`
- `ancient-near-east`
- `mesopotamian-wisdom`
- `presocratics`
- `classical-south-asia`
- `mulamadhyamakakarika`

Capture page labels and workbench nicknames stay in metadata when they are
needed for provenance. Repository paths name the philosophical branch: era,
region, tradition, corpus, archive, inscription, work, figure, concept,
institution, medium, script, source corpus, transmission, controversy,
decipherment route, or local graph route.

AI-generated research packets can suggest branch leads, but they do not witness
source authority. Claims that matter must be anchored later in real source
witnesses, published works, editions, translations, or reviewed canon surfaces.
