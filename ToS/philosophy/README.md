# Philosophy

`ToS/philosophy/` is the growing domain tree of philosophy.

Philosophical branch growth lives here. Temporary extraction routes through
`ToS/candidate-intake/`; this branch grows the authored domain body from trunk
law into eras, regions, traditions, works, figures, concepts, transmissions,
source anchors, and local graph fragments.

## Shape

```text
philosophy/
  trunk/
  eras/
  threads/
  graph-workbench/
```

`trunk/` holds branch-wide law: evidence status, chronology posture, and naming
rules.

`eras/` is the primary growth body. A typical mature branch should grow like:

```text
eras/<era>/
  regions/<region>/
    traditions/<tradition>/
      works/
      figures/
      concepts/
      transmissions/
      sources/
      graph-workbench/
```

`threads/` holds cross-branch navigation for figures, works, concepts, source
corpora, and transmissions. Threads are navigational organs, not the home of
the object.

`graph-workbench/` holds proposed nodes, proposed relations, branch fragments,
and promotion ledgers before any object is raised into `ToS/canon/`.

## Naming

Use names that describe the philosophical topology:

- `bronze-age`
- `ancient-near-east`
- `mesopotamian-wisdom`
- `presocratics`
- `classical-south-asia`
- `mulamadhyamakakarika`

Capture page labels and workbench nicknames stay in metadata when they are
needed for provenance. Repository paths name the philosophical branch: era,
region, tradition, work, figure, concept, source corpus, transmission, or local
graph route.

AI-generated research packets can suggest branch leads, but they do not witness
source authority. Claims that matter must be anchored later in real source
witnesses, published works, editions, translations, or reviewed canon surfaces.
