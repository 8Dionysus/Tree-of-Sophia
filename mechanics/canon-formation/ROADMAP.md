# Canon Formation Roadmap

Canon Formation is planted as the ToS local promotion gate. It protects the
fact that canon is an earned reviewed layer, not a convenient destination.

## Current Contour

- Keep promotion-gate posture local to movement toward `ToS/canon/`.
- Keep source anchors, evidence status, candidate posture, and review state
  visible before anything becomes canonical.
- Keep canonical objects stored in `ToS/canon/`, not mechanics.

## Next Work

- Clarify promotion criteria when branch review and candidate intake start
  producing repeated canon requests.
- Keep promotion residue in the source branch, candidate lane, or canon owner
  surface until a repeated operation needs a mechanics route.

## When Time Comes

- Add payload when promotion ledgers repeatedly outgrow
  `ToS/philosophy/` and `ToS/candidate-intake/` owner routes.
- Split promotion work only if node promotion, relation promotion, and
  vocabulary promotion become distinct repeatable operations.

## Out Of Scope

Canon storage, automatic canonization, and Zarathustra core relocation.
