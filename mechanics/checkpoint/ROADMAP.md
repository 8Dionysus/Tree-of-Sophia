# Checkpoint Roadmap

Checkpoint is planted as review-return posture. It preserves enough state for a
human or agent to resume a ToS route without becoming memory truth, runtime
state, or a second roadmap.

## Current Contour

- Keep review-return as the single current part.
- Keep partial state tied to the ToS owner surface that produced it.
- Keep durable memory claims outside ToS mechanics.
- Keep graph-review, atlas planting, and canon-promotion pauses returnable to
  the source surface that can continue them.

## Next Work

- Clarify return points when review work repeatedly pauses at the same kind of
  witness, branch, candidate, canon, or export boundary.
- Keep return state small enough to resume work without becoming a parallel
  archive.

## When Time Comes

Time has come when: repeated review-return packets need the same shape and can
stay weaker than source, review, and memory owner surfaces.

Move: add a compact review-return packet or split pause, defer, and handoff
only if they become distinct operations.

Guardrail: checkpoint preserves return while durable memory stays with its
stronger owner.

## Out Of Scope

- Durable memory truth.
- Runtime checkpointing.
- Canon storage.
