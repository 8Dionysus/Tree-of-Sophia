# Checkpoint Roadmap

Checkpoint is planted as review-return posture. It preserves enough state for a
human or agent to resume a ToS route without becoming memory truth.

## Current Contour

- Keep review-return as the single current part.
- Keep partial state tied to the ToS owner surface that produced it.
- Keep durable memory claims outside ToS mechanics.

## Next Work

- Clarify return points when review work repeatedly pauses at the same kind of
  witness, branch, candidate, canon, or export boundary.
- Keep return state small enough to resume work without becoming a parallel
  archive.

## When Time Comes

- Add payload when repeated review-return packets need the same shape and can
  stay weaker than source, review, and memory owner surfaces.
- Split review-return only if pause, defer, and handoff become separate
  repeatable operations.

## Out Of Scope

Durable memory truth, runtime checkpointing, and canon storage.
