# Manual Corpus Entry Gate

This document records the current eighth-wave ToS doctrine for manual corpus-entry gating.

The point is simple:
tree motion does not open automatically.

## Core rule

New branch-facing corpus movement should begin from maintainer-led manual source entry.

Templates, review routes, schema hardening, and other soil work may prepare the architecture.
They do not count as actual tree movement by themselves.

## Current explicit gate

The current gate is manual Zarathustra entry.

Until that entry begins, ToS may prepare:

- context-node law
- review law for pre-expansion preparation
- compact relation vocabulary needed by already published node families

It should not quietly treat those preparations as:

- a new family opening
- a second bounded branch
- wider world-thought expansion

## Why this gate matters

Without an explicit gate, the architecture risks confusing:

- preparation with corpus growth
- template publication with source entry
- plurality claims with actual plural material

The gate keeps growth reviewable and honest.
