# Lineage Node Template

This document records the current seventh-wave ToS scaffold for lineage nodes.

The point is not to open a complete lineage ontology.
The point is to give one bounded branch pilot a reviewable lineage-facing form.

## Core fields

A lineage-node scaffold should expose at least:

- `node_id`
- `node_type = lineage`
- `source_anchor`
- `key_terms`
- `distilled_thesis`
- `lineage_relations`
- `interpretation_layers`

The shape stays compact because it reuses the current public node contract.

## Template posture

Use the lineage-node scaffold when:

- the route itself needs a stable authored handle
- a small branch becomes clearer when its movement is named
- lineage is helping orientation rather than decorating it

## Boundary against expansion

One lineage node does not equal a whole branch program.

This template is for a bounded pilot route:

- one family
- one small cluster
- one explicit reviewable path

It does not yet claim a full branch taxonomy or a wider expansion wave.
