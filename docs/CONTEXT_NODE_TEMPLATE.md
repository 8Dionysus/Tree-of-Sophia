# Context Node Template

This document records the current eighth-wave ToS scaffold for context nodes.

The point is not to open a new corpus slice by itself.
The point is to give temporal, geographic, linguistic, civilizational, and institutional context a reviewable authored form before wider expansion begins.

## Core fields

A context-node scaffold should expose at least:

- `node_id`
- `node_type = context`
- `source_anchor`
- `key_terms`
- `distilled_thesis`
- `lineage_relations`
- `interpretation_layers`

The shape stays compact because it reuses the current public node contract.

## Template posture

Use the context-node scaffold when:

- a source or concept needs explicit temporal, geographic, linguistic, civilizational, or institutional framing
- the context itself needs a stable authored handle rather than a passing aside
- context is clarifying source-linked meaning rather than replacing it

## Boundary against premature expansion

Publishing a context-node template does not mean:

- wider world-thought expansion has already begun
- a second family has been opened
- new branch nodes should be added automatically

This is a preparation surface.
It gives future growth a clearer form without pretending the next corpus movement is already underway.
