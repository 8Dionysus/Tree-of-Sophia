# Zarathustra Relation Residue Review

Date: 2026-03-28

## Scope

This note records why the first route-local canonical relation pack for
`thus-spoke-zarathustra/prologue-1` does not promote every row from
`intake/.../edges.csv`.

## Outcome

The canonical relation pack under `tree/relations/` promotes only the 89 edges
whose two endpoints are already canonical ToS surfaces.

The remaining 39 rows stay in `intake/edges.csv` with visible blockers:

- 33 `deferred_residue`
- 3 `deferred_literal`
- 2 `deferred_analogy`
- 1 `deferred_commentary`

## Reason

This deferment is boundary discipline, not silent rejection.

- `deferred_residue` rows still depend on support residues that have not yet
  crossed from intake into canonical tree law.
- `deferred_literal` rows still depend on literal helper surfaces that remain
  intake-only.
- `deferred_analogy` rows still depend on `ev.p5.bee_honey_analogy`, which was
  explicitly held back by family boundary.
- `deferred_commentary` rows still depend on
  `pr.departure_from_reflective_origin`, which remains commentary-like rather
  than distilled principle canon.

The first relation pass therefore keeps canon honest by promoting only fully
settled edges and leaving the rest visibly deferred.
