# Zarathustra Deferred Analogy Review

## Scope

This note records why `ev.p5.bee_honey_analogy` remained in
`intake/thus-spoke-zarathustra/prologue-1/mode-b/event_state_nodes.csv` as
`deferred_analogy` during the route-local `event` and `state` canonization
pass.

## Decision

The row was not rejected for quality.
It was deferred by family boundary.

The current pass canonizes:

- route-local `event` nodes
- route-local `state` nodes

It does not yet open:

- a canonical `analogy` family
- a broader symbolic-reading family

## Rationale

`ev.p5.bee_honey_analogy` is legible and meaningful inside the bounded route,
but it functions as an image-bearing analogy rather than as a direct movement
or held condition.

Promoting it now would blur the newly stabilized distinction between:

- dynamic route movement in `tree/event/`
- dynamic route condition in `tree/state/`
- still-deferred analogy material in `intake/`

Keeping the row visible as `deferred_analogy` preserves the candidate residue
without pretending the family decision has already been made.
