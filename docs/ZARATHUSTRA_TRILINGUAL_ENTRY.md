# Zarathustra Trilingual Entry

This document records the current bounded ToS route for opening manual corpus movement through *Thus Spoke Zarathustra*.

The point is not to open three language trees at once.
The point is to begin one source-first route that stays bounded, reviewable, and honest about language roles.

## Core move

ToS now begins real tree motion through one bounded trilingual entry route inside `tos.source.thus-spoke-zarathustra.prologue`.

The first landed bundle stays deliberately small:

- 2 shared sense blocks from the prologue opening
- 1 shared `node_id`
- 3 language witnesses
- 2 concept companions: `tos.concept.becoming` and `tos.concept.overcoming`

This is a gate-opening route, not wider world-thought expansion.

## Language role contract

The current route fixes asymmetric roles on purpose:

- German is the `canonical_source`
- Russian is the `working_translation`
- English is the `bridge_translation`

German remains the authoritative source witness.
Russian and English remain reviewable witness layers around that source rather than replacement canons.

## Single-tree witness discipline

One meaning should keep one `node_id`.

The current route therefore grows:

- one source node with `language_witnesses`
- shared `segment_id` values across `de / ru / en`
- optional `translation_tensions` only where drift is philosophically load-bearing

It does not grow:

- three language-specific node copies
- multilingual relation tables
- language-specific concept nodes

## First bounded route

The first bounded route uses:

- `tos.source.thus-spoke-zarathustra.prologue`
- `tos.concept.becoming`
- `tos.concept.overcoming`

The chosen 2 sense blocks are the opening threshold of the prologue because they already hold the needed cluster:

- Zarathustra's heart changes
- wisdom turns into overflow
- descent and `untergehen` become active movement rather than defeat

This is enough to test witness alignment without inflating the first landing slice.

## Concept boundary

Concept nodes remain language-neutral even when the source entry is multilingual.

They may point back to a multilingual source route.
They should not become:

- synonym piles
- lexeme registries
- substitute language copies of the source node

## What remains later

This route does not yet open:

- wider world-thought expansion
- cross-tradition multilingual growth
- KAG-facing multilingual projection surfaces
- translation automation

First the bounded trilingual route has to hold under review.
